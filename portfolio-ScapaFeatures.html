Scapa System

Scapa System is a data interpretation and visualisation platform tailored to deliver Towed Streamer EM. At each stage of an EM project, Scapa System provides a set of tools carefully designed for that specific task, from answering feasibility questions, through the planning of an acquisition survey to the QC and visualisation of Towed Streamer EM data.

Date :2004-2014

Role :Project Lead

Client :PGS

Tools:Labview, Matlab, Sketch

Scapa System

As project leader, i enjoyed responsibility for delivering a specialised data visualisation and interpretation platform supporting three generations of a breakthrough technology, from concept to commercialisation, from university start-up to a multinational company.

I was involved across the full scope of the product, with responsibility for planning, mapping and leading toward a successful deployment. With a keen focus on innovative product design to overcome a unique challenge, i was able to deliver solutions that supported the growth and success of a new technology in a rapidly developing industry.

Feasibility

Feasibility and Modelling, includes a tool kit for model building, well log analysis, forward modelling of data - both 1D and 3D - and the inversion of a synthetic data set.

Modelling.

To assess the viability of a prospective acquisition survey, we rigourously investigate likely scenarios based on available information, from well log data to geological interpretations.

Scapa System provides a suite of tools for building complex geo-models, designing survey layout patterns and calculating the sensitivity to target.

Well Logs.

Examine and extract resistivity information from well log analysis, providing a detailed basis for building complex 3D geo-models.

Model Design.

Define region wide geo-models, and design complex survey patterns above target areas to confirm optimum acquisition parameters.

Survey Pattern.

Scapa System allows us to replicate real world scenarios to fully investigate survey parameters prior to acqusition.

Survey Design

Survey Design provides a tool kit for planning the acquisition phase of a survey, from designing layout spreads to defining acquisition parameters.

Pre Survey.

It is important to ensure that we design the most efficient acquisition program possible.

Scapa System allows us to replicate a real world cable design and survey vessel layout, with outputs for generating coverage maps and a complete analysis of prospective acquisition parameters.

Acquisition

Scapa System provides a number of tools based on the analysis of recorded data sets, focused on determining data quality and presenting initial interpreation of results.

Data Attribute.

Scapa System is used on the acquisition vessels to provide an immediate analysis of recorded data, allowing for a near real time overview of an survey area.

Integrated work flows allow for immediate and repeatable actions to be applied to the data, and common agreed products output for each project. These provide critical feedback on data quality and options for fine tuning parameters.

Selection.

Specialised tools for selecting subsets of acquired data to better analyse specific areas of interest.

Survey QC.

A full range of analysis tools to confirm acquisition parameters, and to investigate and correlate any anomalous readings.

Data Plot.

Fully investigate the recorded data set through a number of tailored data plotting options, including viewing channels groups along the survey line.

Interpretation

The Interpreation suite provides a complete tool kit for managing an EM project. From data management, through QC and setting up data for inversion.

Data Response

Tailored data plotting options, focused on viewing large data sets efficiently and completely.

Coverage Plot

Select a subset of acquired data to view, employing the tool kit to ensure we focus on areas of interest.

Line Selection

Select the full survey or a single line, all available within a single UI and options reacting to your selection.

Data Selection

Drill down deeper into the analysis of selected data by highlighting individual points.

Data QC.

Investigate target specific data sets to better understand the relationship between numerous attribute, each of which impact another. The tools are designed to bring these elements together.

Model Building.

Scapa System guides you through the model building process, automating a number of factors that ensure consistency and efficiency across multiple survey lines, projects and teams.

Inversion.

The cumbersome parameterisation of complex inversion is eased by a set of tools tailored to guide and automate where possible. Inversion pathways are highlighted and suggested dependendent on selections in previous steps.

Visualisation

Together with the Intepreation tools, the Visualisation package allows for the QC and plotting of inversion results, through to the creation of the EM product.

Inversion QC.

Acquired EM data is imaged using inversion. The goal of inversion is to estimate a subsurface resistivity model that explains the data, and which is geologically meaningful.

Scapa System has a number of specialised tools for the viewing and interpretation of EM inversion data.

Tools include data comparison, attribute calculation routines and work flows to generate customised EM products for client delivery.

3D Plot.

Custom 3D plotting routines to provide in-depth analysis of generated inversion models. Select sub sets of the model space and investigate iterative changes as the mathematical model developed.

Work Flow.

Automated work flows for converting inversion data sets and generating standard EM products in a variety of industry standard formats.

Attributes.

Delve deeper into the data sets through a set of custom tool kits, providing point by point analysis and the ability to connect together different strands of information.

holoSeis

PGS developed the holoSeis technology in 1999 to become the platform for all seismic processing visualization and QC. Scapa integrates with holoSeis to provide additional visualisation and interpretation opportunities, with a key focus on the integration of EM and Seismic data sets.

Automation

Scapa System provides a wrapper interface which facilities connections between individual components and encourages the automation of standardised work flows.

JSON.

Automation begins with a single parameter file, defined in the JSON format. This single file, in conjunction with a set of MATLAB library calls, controls the flow and selection of data through a standard EM work flow.

The underlying processing routines developed in Scapa System are made visible to the MATLAB scripting language. A sequence of standard work flow patterns have been developed, each of which are easily tailored with project specific parameters.
